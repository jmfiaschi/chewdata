Get started

Without Garbage Collector

This ETL doesn't use any garbage collector for better stability. Everything is handle by the code in Rust.

Versionable

The configuration can be versionable and give the capability to retrieve old version with a version control system.

Scalable and Parallelizable

The application is a simple job that can be executed a multiple time in different context.

Simple command executable everywhere

Choose your serveur where you want to execute the etl : Local, Kube, AWS Lambda, AWS EC2, etc...

Customize your package

With rust's feature flags, enable only required componants.

cargo install chewdata --no-default-features --features "use_bucket_connector use_curl_connector"

Easy to install

cargo install chewdata
echo '{"field":"value"}' | chewdata
>> [{"field":"value"}]
